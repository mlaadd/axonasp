Welcome to the AxonASP Configuration Manager. This utility provides an interactive interface to easily manage and modify the configuration settings for your AxonASP server.

While this tool manages the configuration file directly, remember that you can always override these settings dynamically by using environment variables. To do so, use the same name as the setting you wish to override, formatted in uppercase with dots replaced by underscores. For example, to override the default_charset setting, you would set an environment variable named DEFAULT_CHARSET. This flexibility allows you to easily manage different configurations across development and production environments. Make sure to restart the AxonASP server after saving changes here or modifying environment variables for the new settings to take effect.

Important Saving Instructions: When navigating through this manager, you must save your changes before leaving each section page. If you navigate to another page without saving, any edits made within that specific section will be lost.

Command-Line Options:

By default, this program will load and work with the standard configuration file. However, you can modify the utility's behavior by launching axonadmin from your terminal with the following flags:

-edit <path> — Specifies a custom TOML configuration file to edit. If omitted, the utility mimics the standard loader path resolution.
-create <path> — Generates a new default configuration file at the specified path. If the path is left empty, it defaults to ./config/axonasp.toml.
-noui or -w — Runs the program in headless mode (note: this must be used in conjunction with the -create flag).
-h or -help — Displays the help menu in your terminal.

Backups and Restoration:

To protect against accidental misconfigurations that could affect server stability and security, this program automatically safeguards your settings. Every time you save a change, a backup of the previous TOML configuration is generated using the same filename but with a .bak extension. This allows you to quickly roll back to the previous setup if necessary. Furthermore, you can always restore the original, factory-default AxonASP configuration file at any time should you need a clean slate.
